# Git and GitHub

## What is Git?

Git is a `version control` tool.

What is version control?

Let's say you're writing a document on your computer, perhaps writing something today and something else another day. At some point, you might feel that what you wrote in the last two days wasn't right, and you want to delete it and go back to how it was two days ago. In this case, you would need to delete all the writing from these two days. Now, if you can't remember exactly how much to delete or how much was written two days ago, what would you do? This is where Git helps. Git monitors or tracks every change in a file and versions each change. This allows you to switch to any version at any time.

## Some essential Git commands that we'll need to manage files in our local Git repository.

1. **Initialize Git**

```bash
 git init
```

2. **Check Git Status**

```bash
git status
```

3. **Stage files for Git tracking**

```bash
git add <filename> //For stage any specific file
or
git add --all // if you want to add all files
or
git add .  // if you want to add all files in the current working directory
```

4. **Commit changes**

```bash
git commit -m "<commit massage>"
```

5. **Check Git log**

```bash
git log
or
git log --oneline
```

6. **Go back from one commit to another commit state**

```bash
git reset --hard <commit id>
```

7. **View Git log with references to all changes**

```bash
git reflog
```

8. **Remove a file from Git tracking**

```bash
git rm <filename>
```

9. **Create a branch**

```bash
git branch <branch name eg: bugfix/fixname >
```

10. **View branch list**

```bash
git branch --list
```

11. **Switch from one branch to another**

```bash
git switch <branch name >
```

12. **Merge code from one branch to another**

  <Callout type='warning' emoji="💡">
    In this case, you need to first switch to the branch where you want to bring in code from another branch using `git switch <branchname>`. Then you can run the command.
</Callout>

```bash
git merge <branch name >
```

13. **Delete a branch from Git**

```bash
git branch -d <branch name >
```

14. **To change a branch name**

 <Callout type='warning' emoji="💡"> In this case, we need to first switch to the branch whose name we want to change using `git switch <branchname>`. Then we can run the command. </Callout>

```bash
git branch -m <updated branch name >
```

15. **Stash Git changes (save uncommitted work in Git's draft)**

```bash
git stash
```

16. **Check Git stash list**

```bash
git stash list
```

17. **To see what was saved in Git's stash**

```bash
git stash show -p
```

18. **To apply Git stash**

```bash
git stash apply <statsh id>
or
git stash pop
```

19. If you accidentally add a file to Git or after committing you feel that the file should be ignored from Git, then:

<Steps>

### Step 1:

Create a .gitignore file and add the name of that file to it.

### Step 2:

```bash
git rm --cached <filename>
```

</Steps>

20. To add or push files to GitHub or a remote repository:

<Steps>
### Step 1:

```bash
git remote add <origin name eg: orgin or something > <github repository link>
```

example:

`git remote add origin git@github.com:Deveripon/reactive-accelarator.git`

### Step 2:

```bash
git push -u <origin name> <branch name>
```

example:

`git push -u origin main`

</Steps>

21. Pull from a remote repository

```bash
git pull
```
