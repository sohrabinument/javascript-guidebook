# Git and GitHub

## What is Git?

Git is a `version control` tool.

What is version control?

Imagine you're writing a document on your computer, writing something today and something else on another day. At some point, you might feel that what you wrote in the last two days is not correct, and you want to delete it and go back to the state of two days ago. In this case, you would need to delete all of what you wrote in these two days. Now, if you can't remember exactly how much to delete or how much was written two days ago, what would you do? This is where Git helps. Git monitors or tracks every change in a file and versions each change. This allows you to switch to any version at any time.

## Some essential Git commands that we'll need to manage files in our local Git repository.

1. **Initialize Git**

```bash
 git init
```

2. **Check Git status**

```bash
git status
```

3. **Stage files for Git tracking**

```bash
git add <filename> //For stage any specific file
or
git add --all // if you want to add all files
or
git add .  // if you want to add all files in the current working directory
```

4. **Commit changes**

```bash
git commit -m "<commit massage>"
```

5. **Check Git log**

```bash
git log
or
git log --oneline
```

6. **Go back from one commit state to another**

```bash
git reset --hard <commit id>
```

7. **View Git log with references for each change**

```bash
git reflog
```

8. **Remove a file from Git tracking**

```bash
git rm <filename>
```

9. **Create a branch**

```bash
git branch <branch name eg: bugfix/fixname >
```

10. **View branch list**

```bash
git branch --list
```

11. **Switch from one branch to another**

```bash
git switch <branch name >
```

12. **Merge code from one branch to another**

  <Callout type='warning' emoji="💡">
    In this case, to bring code from another branch into the branch you want, first use `git switch <branchname>` to go to that branch. Then enter the command.
</Callout>

```bash
git merge <branch name >
```

13. **Delete a branch from Git**

```bash
git branch -d <branch name >
```

14. **Change branch name**

 <Callout type='warning' emoji="💡"> In this case, to change the name of a branch, first use `git switch <branchname>` to go to that branch. Then enter the command. </Callout>

```bash
git branch -m <updated branch name >
```

15. **Stash Git changes (save uncommitted work in Git draft)**

```bash
git stash
```

16. **Check stash list**

```bash
git stash list
```

17. **See what was saved in Git stash**

```bash
git stash show -p
```

18. **Apply Git stash**

```bash
git stash apply <statsh id>
or
git stash pop
```

19. If you accidentally add a file to Git or realize after committing that you want to ignore it from Git:

<Steps>

### Step 1:

Create a .gitignore file and add the name of that file to it.

### Step 2:

```bash
git rm --cached <filename>
```

</Steps>

20. To add or push files to GitHub or a remote repository:

<Steps>
### Step 1:

```bash
git remote add <origin name eg: orgin or something > <github repository link>
```

example:

`git remote add origin git@github.com:Deveripon/reactive-accelarator.git`

### Step 2:

```bash
git push -u <origin name> <branch name>
```

example:

`git push -u origin main`

</Steps>

21. Pull from remote repository

```bash
git pull
```
