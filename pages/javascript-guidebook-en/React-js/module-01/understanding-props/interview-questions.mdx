# Interview Questions: Understanding Props in React

## Basic

**Q1: What are props in React?**  
A: Props are inputs to a React component that allow data to be passed from a parent component to a child component.

**Q2: How do you pass a prop to a component?**  
A: By adding attributes to the component in JSX, e.g., `<Avatar src="avatar.png" />`.

---

## Intermediate

**Q3: How do you set a default value for a prop?**  
A: By using destructuring with a default value, e.g., `function Avatar({ size = 100 }) { ... }`.

**Q4: Can a child component change its own props?**  
A: No, props are read-only in the child component. Only the parent can change the value and pass a new prop.

**Q5: What is the `children` prop in React?**  
A: The `children` prop is a special prop that contains any nested JSX or components inside a component tag.

---

## Advanced / Scenario-based

**Q6: What happens if you pass `undefined`, `null`, or `false` to a prop with a default value?**  
A: The default value is only used if the prop is `undefined` or not provided. If you pass `null` or `false`, those values will be used instead of the default.

**Q7: How can you forward all props from one component to another?**  
A: By using the spread syntax: `<Avatar {...props} />`.

**Q8: Is it possible to pass a function as a prop? Give an example.**  
A: Yes. Example: `<Button onClick={handleClick} />` where `handleClick` is a function.

**Q9: Why should you use the spread syntax for props with care?**  
A: Because it can make your code less explicit and harder to understand which props are being passed.
