# Importing and Exporting Components in React

React's component system is powerful because it lets you split your UI into reusable pieces. As your app grows, you'll want to organize components into separate files and import/export them as needed.

## Why Split Components into Files?

- **Organization:** Easier to find and maintain code.
- **Reusability:** Use the same component in multiple places.
- **Scalability:** Large apps are easier to manage.

## Exporting a Component

There are two main ways to export a component:

### 1. Default Export

Use `export default` if your file exports only one main component.

```jsx
// Profile.js
export default function Profile() {
  return <img src="https://i.imgur.com/MK3eW3Am.jpg" alt="Katherine Johnson" />;
}
```

### 2. Named Export

Use `export` (without default) if you want to export multiple components from the same file.

```jsx
// Gallery.js
export function Profile() {
  return <img src="https://i.imgur.com/MK3eW3Am.jpg" alt="Katherine Johnson" />;
}

export function Gallery() {
  return (
    <section>
      <h1>Amazing scientists</h1>
      <Profile />
      <Profile />
      <Profile />
    </section>
  );
}
```

## Importing a Component

How you import depends on how you exported:

### Importing a Default Export

```jsx
import Profile from './Profile';
```

### Importing a Named Export

```jsx
import { Profile, Gallery } from './Gallery';
```

> **Note:** You can only have one default export per file, but as many named exports as you want.

## Mixing Default and Named Exports

You can mix both in one file, but be careful with the import syntax:

```jsx
// Gallery.js
export function Profile() { /* ... */ }
export default function Gallery() { /* ... */ }

// App.js
import Gallery, { Profile } from './Gallery';
```

## Best Practices

- Use default export for the main component in a file.
- Use named exports for helper or additional components.
- Always use meaningful names for your components and files.
- Be consistent with your export/import style.

## Example: Splitting Components

Suppose you have a `Gallery` and a `Profile` component. You can put them in separate files:

**Profile.js**
```jsx
export default function Profile() {
  return <img src="https://i.imgur.com/MK3eW3Am.jpg" alt="Katherine Johnson" />;
}
```

**Gallery.js**
```jsx
import Profile from './Profile';

export default function Gallery() {
  return (
    <section>
      <h1>Amazing scientists</h1>
      <Profile />
      <Profile />
      <Profile />
    </section>
  );
}
```

**App.js**
```jsx
import Gallery from './Gallery';

export default function App() {
  return <Gallery />;
}
```

For more, see the [official React documentation on importing and exporting components](https://react.dev/learn/importing-and-exporting-components).
