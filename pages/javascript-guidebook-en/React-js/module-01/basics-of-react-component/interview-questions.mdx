# Basics of React Component – Interview Questions & Answers

Use these questions to prepare for interviews or test your understanding of React components.

---

**Q1: What is a React component?**  
A: A reusable, self-contained piece of UI in React, defined as a function or class.

**Q2: Why should component names start with a capital letter?**  
A: React treats lowercase tags as HTML elements and capitalized tags as custom components.

**Q3: How do you create a function component in React?**  
A: By defining a JavaScript function that returns JSX and starts with a capital letter.

**Q4: How do you use one component inside another?**  
A: By importing it and using it as a custom tag, e.g., `<Profile />`.

**Q5: What are the benefits of using components in React?**  
A: Reusability, separation of concerns, easier maintenance, and better organization.

**Q6: Can you nest function definitions inside a component? Why or why not?**  
A: No, you should not nest function definitions inside a component. Instead, define helper functions outside or as separate components to keep code clean and maintainable.

**Q7: Scenario: You have a button that is used in multiple places with different labels. How would you design this as a reusable component?**  
A: Create a Button component that accepts a label prop and use it wherever needed, passing different labels as props.

**Q8: What happens if you forget to capitalize a component name?**  
A: React will treat it as a regular HTML tag, and it will not render your component as expected.

**Q9: When should you split a component into multiple files?**  
A: When the component is large, complex, or reused in multiple places.

**Tip:** Review these questions and try to explain the concepts in your own words for the best interview preparation!
