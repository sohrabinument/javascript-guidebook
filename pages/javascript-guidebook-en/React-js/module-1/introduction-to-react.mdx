## What is `React`?

React is a JavaScript library developed by Facebook. It helps design both native and web interfaces. It is a JavaScript library written in JavaScript.

Facebook software engineer Jordan Walke created React.js in 2011, inspired by XHP-js, a component-based framework for PHP. In 2011, it was first implemented in Facebook's news feed, and in 2012, it was used in Instagram. In 2013, Facebook open-sourced React.js. React's open-source GitHub repository [React.js](https://github.com/facebook/react) contains nothing but vanilla JavaScript.

## Why should we use `React`, what's the problem with vanilla JavaScript?

When we work with vanilla JavaScript, what happens is that when UI changes are needed due to user interactions, we have to manually make each UI change. But in `React`, we don't need to do that. In React, whenever data or state changes due to user interaction, the UI automatically reacts and changes automatically. This process of repeatedly updating the UI is what React was primarily created to automate.

## What are `React` and `React DOM`?

To build a UI, three aspects need to be managed:

-   1. Creating the UI/DOM.
-   2. Handling user interactions.
-   3. Rendering the DOM in the browser.

The `React` and `React DOM` packages handle these tasks.

`React` helps build the USER INTERFACE and manages changes in data that occur due to user interactions.

The `React DOM` package renders the DOM in the browser, and once rendered, it waits for data changes. If any data changes anywhere, it identifies that change in its `VIRTUAL DOM`, makes the necessary UI changes, and directly paints them in the browser. This allows the user to see the updated UI.
