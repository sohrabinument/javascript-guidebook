## What are `Components`?

`Component` is one of the core concepts of React. It is called the building block of UI. Each part of the UI code that can be reused repeatedly can be considered a component. Even a small button can be a component.

React helps us write our markup in small reusable code blocks, each block being a component.

However, there are some rules for creating components, which are shown below.

## How to create a React `Component`?

A React component is a JavaScript function that combines markup with JavaScript.

**React Component Syntax**

```javascript showLineNumbers filename="profile.jsx"
export default function Profile() {
    return (
        <img
            src='https://i.imgur.com/MK3eW3Am.jpg'
            alt='Katherine Johnson'
        />
    );
}
```

**Steps to follow when creating a component:**

<Steps>
    ### Export the component

    Use `export default` to make a component reusable, so we can `import` the component in other files.

    ### Define the function
    After `export default`, define a named function.
    ```javascript showlineNumbers
        export default function Profile(){}
    ```

    <Callout type="warning" emoji='💡'>
        When creating a `React Component`, you must write the first letter of the function name with a `Capital letter`, otherwise it won't work.
    </Callout>

    ### Add markup inside the function

    Return markup in `JSX (Javascript XML)` syntax inside the function.

    ```javascript showLineNumbers {2}
    export default function Profile() {
        return img src="https://i.imgur.com/MK3eW3As.jpg" alt="Katherine Johnson" />
    }
    ```
    **If the `return statement` needs to be written in more than one line, it must be written inside `()`**
    Example:
      ```javascript showLineNumbers filename="Profile.jsx" {2-6}
    export default function Profile() {
        return (
        <div>
             <img src="https://i.imgur.com/MK3eW3As.jpg" alt="Katherine Johnson" />
        </div>
        )
    }
    ```

</Steps>

## How to use a component:

After creating a component, it can be used inside another component in the same file or by `importing` the component in another file.

Example:

```javascript showLineNumbers filename="Gallery.jsx"
import Profile from "Profile";
export default function Gallery() {
    return (
        <section>
            <h1>Amazing scientists</h1>
            <Profile />
            <Profile />
            <Profile />
        </section>
    );
}
```

## Rules to follow when nesting components:

**`Rule 1` - If a component is very small and won't be reused, it can be written and used in the same file. But if the component is large, has more `interactivity`, and needs to be reused in different places, then components should be created in separate files.**

**`Rule 2` - You can nest one component inside another component, but you can't nest one function definition inside a component.**
