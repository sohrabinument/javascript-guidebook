## 1.1 – Event Handling Fundamentals 🖱️

<div style={{ background: 'rgba(0, 100, 200, 0.10)', padding: '15px', borderRadius: '5px', marginBottom: '20px', border: '1px solid rgba(0, 100, 200, 0.15)', color: 'inherit' }}>

**Q1: What is an event handler in React?**  
A: An event handler is a function that runs in response to user actions (like clicks, typing, or form submissions) in your UI. In React, you attach event handlers as props (e.g., `onClick`, `onChange`).

**Q2: How do you add a click event handler to a button in React?**  
A: By passing a function reference to the `onClick` prop:
```jsx
<button onClick={handleClick}>Click me</button>
```

**Q3: Why should you pass a function reference (not a function call) to event props?**  
A: Passing a reference (e.g., `onClick={handleClick}`) ensures the function runs only when the event occurs. If you call it directly (e.g., `onClick={handleClick()}`), it runs immediately on render.

## 1.2 – Inline Handlers & Best Practices 💡

<div style={{ background: 'rgba(230, 150, 70, 0.10)', padding: '15px', borderRadius: '5px', marginBottom: '20px', border: '1px solid rgba(230, 150, 70, 0.15)', color: 'inherit' }}>

**Q4: When should you use an inline event handler?**  
A: Use inline handlers for quick, simple actions (e.g., `onClick={() => alert('Hi!')}`). For complex logic, define a separate function.

**Q5: What happens if you call the handler function directly in the prop?**  
A: The function will run every time the component renders, not just when the user interacts.

**Q6: How do you prevent the default form submission behavior in React?**  
A: Call `e.preventDefault()` inside your handler:
```jsx
function handleSubmit(e) {
  e.preventDefault();
  // custom logic
}
```

## 1.3 – Scenario & Practical Example 🏆

### Scenario-Based Question:

You have a form with an input and a submit button. You want to show an alert with the input value when the form is submitted, but prevent the page from reloading.

**Question:** How would you implement this in React? What best practices would you follow?

Take a moment to formulate your answer before checking the suggested response.

**Sample Approach:**

```jsx
import { useState } from 'react';

export default function MyForm() {
  const [value, setValue] = useState('');
  const handleSubmit = (e) => {
    e.preventDefault();
    alert(value);
  };
  return (
    <form onSubmit={handleSubmit}>
      <input value={value} onChange={e => setValue(e.target.value)} />
      <button type="submit">Submit</button>
    </form>
  );
}
```

- Use `e.preventDefault()` to stop the default behavior.
- Use state to track the input value.
- Use a named handler for clarity.
