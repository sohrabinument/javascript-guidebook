**Q1: What is state in React?**  
A: State is a way for React components to "remember" information between renders. It lets components update and display dynamic data.

**Q2: How do you declare a state variable in a function component?**  
A: By using the `useState` Hook, e.g. `const [count, setCount] = useState(0);`

**Q3: What does the setter function returned by useState do?**  
A: It updates the state variable and triggers a re-render of the component.

**Q4: Where can you call the useState Hook?**  
A: Only at the top level of a React function component or a custom Hookâ€”not inside loops, conditions, or nested functions.

**Q5: What happens if you call useState inside a loop or condition?**  
A: It can break the rules of Hooks, leading to bugs and unpredictable behavior.

**Q6: Is state shared between different instances of a component?**  
A: No, each instance of a component has its own isolated state.

### Scenario-Based Question:

Suppose you have a counter component with `const [count, setCount] = useState(0);`. What happens if you call `setCount(count + 1)` twice in a row inside the same function?

**Sample Answer:**
- Both calls will use the same value of `count` (the current state), so the counter will only increase by 1, not 2.
- To increment by 2, use the updater function: `setCount(c => c + 1); setCount(c => c + 1);`
