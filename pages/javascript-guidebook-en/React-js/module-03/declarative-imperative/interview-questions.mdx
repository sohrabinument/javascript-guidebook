## 1.1 ‚Äì Fundamentals üß©

<div style={{ background: 'rgba(0, 100, 200, 0.10)', padding: '15px', borderRadius: '5px', marginBottom: '20px', border: '1px solid rgba(0, 100, 200, 0.15)', color: 'inherit' }}>

**Q1: What is imperative programming in the context of UI?**  
A: Imperative programming is when you tell the computer step-by-step how to update the UI, specifying each instruction in detail (e.g., show/hide elements, change styles directly).

**Q2: What is declarative programming in the context of UI?**  
A: Declarative programming is when you describe what the UI should look like for a given state, and the framework (like React) figures out how to update the UI accordingly.

**Q3: Which approach does React use?**  
A: React uses a declarative approach, letting you describe the UI for each state and handling the updates automatically.

## 1.2 ‚Äì Rules & Pitfalls ‚ö†Ô∏è

**Q4: What is a common pitfall of imperative UI programming?**  
A: It can become hard to manage and error-prone as the UI grows, since you must manually update every part of the UI, increasing the risk of bugs.

**Q5: Why is declarative programming often preferred for complex UIs?**  
A: Because it makes code easier to read, maintain, and reason about‚ÄîReact takes care of updating the UI based on state changes.

**Q6: Can you mix imperative and declarative code in React?**  
A: Yes, but it's best to stay declarative as much as possible. Sometimes, for direct DOM manipulation, you might use imperative code (e.g., with refs).

### Scenario-Based Question:

Suppose you want to show a loading spinner when a form is submitting, and hide it when done. How would you do this imperatively vs declaratively?

Take a moment to think before checking the answer.

<details>
<summary>**Click to see a sample answer**</summary>

**Imperative:**
- You would manually show/hide the spinner by changing its style or class in response to events (e.g., `spinner.style.display = 'block'`).

**Declarative (React):**
- You would use a state variable like `isLoading`, and conditionally render the spinner: `{isLoading && <Spinner />}`. React handles showing/hiding based on state.
