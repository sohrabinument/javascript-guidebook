# কন্ডিশনাল রেন্ডারিং ইন React: ইন্টারভিউ প্রশ্ন ও উত্তর

1. **React-এ কন্ডিশনাল রেন্ডারিং কী?**  
   **উত্তর:** React-এ কন্ডিশনাল রেন্ডারিং হলো নির্দিষ্ট কোনো শর্ত (যেমন state বা props) অনুযায়ী বিভিন্ন UI এলিমেন্ট বা কম্পোনেন্ট দেখানোর প্রক্রিয়া। এর মাধ্যমে আপনি আপনার অ্যাপ্লিকেশনের লজিক অনুযায়ী DOM-এ কী রেন্ডার হবে তা নিয়ন্ত্রণ করতে পারেন।

2. **JSX-এ কিভাবে কন্ডিশনাল রেন্ডারিং ইমপ্লিমেন্ট করা যায়?**  
   **উত্তর:** JSX-এ কন্ডিশনাল রেন্ডারিং করতে পারেন if/else স্টেটমেন্ট, টার্নারি অপারেটর (`condition ? expr1 : expr2`), এবং লজিক্যাল AND (`&&`) অপারেটর ব্যবহার করে। যেমন:

   ```jsx
   {
     isLoggedIn ? <Dashboard /> : <Login />;
   }
   ```

3. **React কম্পোনেন্টে if/else এবং টার্নারি অপারেটরের মধ্যে পার্থক্য কী?**  
   **উত্তর:** if/else স্টেটমেন্ট সাধারণত JSX-এর বাইরে ব্যবহার করা হয় এবং কোন JSX রিটার্ন হবে তা নির্ধারণ করে। টার্নারি অপারেটর সরাসরি JSX-এর মধ্যে ব্যবহার করা যায় এবং এটি ইনলাইন কন্ডিশনাল রেন্ডারিং-এর জন্য বেশি উপযোগী ও সংক্ষিপ্ত।

4. **React-এ কন্ডিশনাল রেন্ডারিং-এর জন্য লজিক্যাল AND (`&&`) অপারেটর কীভাবে কাজ করে?**  
   **উত্তর:** লজিক্যাল AND (`&&`) অপারেটরটি বাম পাশের শর্তটি সত্য হলে ডান পাশের এক্সপ্রেশনটি রেন্ডার করে। শর্তটি মিথ্যা হলে কিছুই রেন্ডার হয় না। যেমন:

   ```jsx
   {
     hasUnread && <p>আপনার অপঠিত মেসেজ আছে!</p>;
   }
   ```

5. **React কম্পোনেন্ট থেকে `null` রিটার্ন করা যায় কি? এটা কী করে?**  
   **উত্তর:** হ্যাঁ, React কম্পোনেন্ট থেকে `null` রিটার্ন করা যায়। এতে করে ঐ কম্পোনেন্টের জন্য DOM-এ কিছুই রেন্ডার হয় না।

6. **কন্ডিশনাল রেন্ডারিং কোড রিডেবল রাখার জন্য কিছু বেস্ট প্র্যাকটিস কী?**  
   **উত্তর:**

   - সহজ কন্ডিশনের জন্য টার্নারি অপারেটর ব্যবহার করুন।
   - জটিল লজিকের জন্য if/else বা ভ্যারিয়েবল ব্যবহার করুন।
   - গভীরভাবে নেস্টেড টার্নারি অপারেটর এড়িয়ে চলুন।
   - জটিল কন্ডিশনাল লজিক আলাদা ফাংশন বা কম্পোনেন্টে নিয়ে যান।

7. **কিভাবে একটি লিস্ট খালি না হলে কেবলমাত্র একটি কম্পোনেন্ট রেন্ডার করবেন?**  
   **উত্তর:** লজিক্যাল AND (`&&`) অপারেটর ব্যবহার করে লিস্টে আইটেম থাকলে কম্পোনেন্ট রেন্ডার করতে পারেন:
   ```jsx
   {
     items.length > 0 && <ItemList items={items} />;
   }
   ```
