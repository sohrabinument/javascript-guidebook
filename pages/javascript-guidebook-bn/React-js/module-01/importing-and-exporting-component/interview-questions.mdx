# কম্পোনেন্ট ইম্পোর্ট ও এক্সপোর্ট – ইন্টারভিউ প্রশ্ন ও উত্তর

এই প্রশ্নোত্তরগুলো রিয়্যাক্টে কম্পোনেন্ট ইম্পোর্ট ও এক্সপোর্ট সম্পর্কে ইন্টারভিউ প্রস্তুতি বা নিজের ধারণা যাচাইয়ের জন্য ব্যবহার করুন।

---

**প্রশ্ন ১: কেন কম্পোনেন্ট আলাদা ফাইলে রাখা উচিত?**  
উত্তর: কোডের সংগঠন, পুনঃব্যবহারযোগ্যতা ও রক্ষণাবেক্ষণ সহজ করার জন্য।

**প্রশ্ন ২: রিয়্যাক্টে ডিফল্ট ও নেমড এক্সপোর্টের মধ্যে পার্থক্য কী?**  
উত্তর: ডিফল্ট এক্সপোর্টে প্রতি ফাইলে একটি এক্সপোর্ট থাকে এবং যেকোনো নামে ইম্পোর্ট করা যায়। নেমড এক্সপোর্টে একাধিক এক্সপোর্ট রাখা যায় এবং নির্দিষ্ট নামেই ইম্পোর্ট করতে হয়।

**প্রশ্ন ৩: ডিফল্ট এক্সপোর্ট কীভাবে ইম্পোর্ট করবেন?**  
উত্তর: `import ComponentName from './ComponentFile';`

**প্রশ্ন ৪: নেমড এক্সপোর্ট কীভাবে ইম্পোর্ট করবেন?**  
উত্তর: `import { ComponentName } from './ComponentFile';`

**প্রশ্ন ৫: একই ফাইলে ডিফল্ট ও নেমড এক্সপোর্ট রাখা যায় কি? ইম্পোর্ট করবেন কীভাবে?**  
উত্তর: হ্যাঁ। ডিফল্ট এক্সপোর্ট ব্রেস ছাড়া, নেমড এক্সপোর্ট ব্রেসসহ ইম্পোর্ট করতে হয়:  
`import DefaultComponent, { NamedComponent } from './ComponentFile';`

**প্রশ্ন ৬: ডিফল্ট এক্সপোর্টকে নেমড ইম্পোর্ট বা নেমড এক্সপোর্টকে ডিফল্ট ইম্পোর্ট করলে কী হয়?**  
উত্তর: এরর হবে; ইম্পোর্ট/এক্সপোর্ট সিনট্যাক্স মিলতে হবে।

**প্রশ্ন ৭: পরিস্থিতি: আপনি একই ফাইল থেকে `Gallery` ও `Profile` ব্যবহার করতে চান। কীভাবে এক্সপোর্ট ও ইম্পোর্ট করবেন?**  
উত্তর: একটিকে ডিফল্ট, অন্যটিকে নেমড এক্সপোর্ট করুন, বা দুটোই নেমড এক্সপোর্ট করুন। ইম্পোর্ট:  
`import Gallery, { Profile } from './Gallery';` অথবা `import { Gallery, Profile } from './Gallery';`

**প্রশ্ন ৮: অ্যানোনিমাস ডিফল্ট এক্সপোর্ট (যেমন `export default () => {}`) ব্যবহার নিরুৎসাহিত কেন?**  
উত্তর: এতে ডিবাগিং কঠিন হয়, কারণ কম্পোনেন্টের অর্থবহ নাম থাকে না, ফলে এরর মেসেজ বা React DevTools-এ খুঁজে পাওয়া কঠিন হয়।

---

**টিপ:** ছোট React প্রজেক্টে কম্পোনেন্ট ভাগ ও ইম্পোর্ট-এক্সপোর্ট প্র্যাকটিস করুন—এতে ধারণা আরও পরিষ্কার হবে!
