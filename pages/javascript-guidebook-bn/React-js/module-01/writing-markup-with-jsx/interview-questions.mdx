# JSX ইন্টারভিউ প্রশ্ন ও উত্তর

## বেসিক

**প্রশ্ন ১: JSX কী?**  
উত্তর: JSX হলো জাভাস্ক্রিপ্টের জন্য একটি সিনট্যাক্স এক্সটেনশন, যা React কম্পোনেন্টে HTML-এর মতো মার্কআপ লিখতে দেয়।

**প্রশ্ন ২: React-এ আমরা কেন JSX ব্যবহার করি?**  
উত্তর: JSX ব্যবহার করলে UI স্ট্রাকচার লেখা ও দেখা সহজ হয় এবং মার্কআপ ও রেন্ডারিং লজিক একসাথে রাখা যায়।

**প্রশ্ন ৩: JSX ছাড়া কি React কম্পোনেন্ট লেখা যায়?**  
উত্তর: হ্যাঁ, `React.createElement` ব্যবহার করে লেখা যায়, তবে JSX বেশি সংক্ষিপ্ত ও পড়তে সহজ।

---

## ইন্টারমিডিয়েট

**প্রশ্ন ৪: HTML ও JSX-এর মধ্যে কিছু পার্থক্য কী?**  
উত্তর: JSX-এ `class` এর পরিবর্তে `className`, সব ট্যাগ ক্লোজ করতে হয় এবং বেশিরভাগ অ্যাট্রিবিউটে camelCase ব্যবহার করতে হয়।

**প্রশ্ন ৫: JSX-এ কীভাবে জাভাস্ক্রিপ্ট এক্সপ্রেশন এম্বেড করা যায়?**  
উত্তর: কার্লি ব্রেস `{}` দিয়ে এক্সপ্রেশন এম্বেড করা যায়।

## অ্যাডভান্সড / সিনারিও-ভিত্তিক

**প্রশ্ন ৬: যদি আপনি একাধিক JSX এলিমেন্ট প্যারেন্ট ছাড়া রিটার্ন করেন, কী হবে?**  
উত্তর: React একটি এরর দিবে। অবশ্যই একটি প্যারেন্ট এলিমেন্ট বা Fragment (`<>...</>`) দিয়ে র‍্যাপ করতে হবে।

**প্রশ্ন ৭: নিচের HTML-কে বৈধ JSX-এ রূপান্তর করুন:**

```html
<p class="info">Hello <b>World</b></p>
```

উত্তর:

```jsx
<p className="info">
  Hello <b>World</b>
</p>
```

**প্রশ্ন ৮: JSX-এ কিছু অ্যাট্রিবিউটে camelCase কেন ব্যবহার করতে হয়?**  
উত্তর: JSX জাভাস্ক্রিপ্টে কম্পাইল হয় এবং জাভাস্ক্রিপ্ট অবজেক্টের কী-তে ক্যামেলকেস ও রিজার্ভড ওয়ার্ড এড়াতে হয়, যেমন `class`।

**প্রশ্ন ৯: JSX-এ Fragment কী এবং কখন ব্যবহার করবেন?**  
উত্তর: Fragment (`<>...</>`) ব্যবহার করলে একাধিক এলিমেন্ট একসাথে রিটার্ন করা যায়, কিন্তু DOM-এ বাড়তি কোনো নোড যোগ হয় না।

**প্রশ্ন ১০: HTML থেকে JSX-এ রূপান্তরের সময় সাধারণ কিছু ভুল কী কী?**  
উত্তর: ট্যাগ ক্লোজ না করা, `class` ব্যবহার করা, ক্যামেলকেস না করা, এবং প্যারেন্ট ছাড়া একাধিক এলিমেন্ট রিটার্ন করা।
