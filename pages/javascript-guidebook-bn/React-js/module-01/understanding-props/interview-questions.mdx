# ইন্টারভিউ প্রশ্নাবলী: React-এ Props বোঝা

## বেসিক

**প্রশ্ন ১: React-এ props কী?**  
উত্তর: Props হলো React কম্পোনেন্টের ইনপুট, যা প্যারেন্ট কম্পোনেন্ট থেকে চাইল্ড কম্পোনেন্টে ডেটা পাঠাতে ব্যবহৃত হয়।

**প্রশ্ন ২: কিভাবে একটি কম্পোনেন্টে prop পাঠানো হয়?**  
উত্তর: JSX-এ কম্পোনেন্টে অ্যাট্রিবিউট হিসেবে prop পাঠানো হয়, যেমন: `<Avatar src="avatar.png" />`।

---

## ইন্টারমিডিয়েট

**প্রশ্ন ৩: কিভাবে prop-এর জন্য ডিফল্ট ভ্যালু সেট করা যায়?**  
উত্তর: Destructuring-এর সময় ডিফল্ট ভ্যালু ব্যবহার করে, যেমন: `function Avatar({ size = 100 }) { ... }`

**প্রশ্ন ৪: চাইল্ড কম্পোনেন্ট কি নিজের props পরিবর্তন করতে পারে?**  
উত্তর: না, props শুধুমাত্র পড়ার জন্য। শুধু প্যারেন্ট কম্পোনেন্টই prop-এর মান পরিবর্তন করে নতুন prop পাঠাতে পারে।

**প্রশ্ন ৫: React-এ `children` prop কী?**  
উত্তর: `children` prop হলো একটি বিশেষ prop, যা কম্পোনেন্ট ট্যাগের মধ্যে নেস্ট করা JSX বা কম্পোনেন্ট ধারণ করে।

---

## অ্যাডভান্সড / সিনারিও-ভিত্তিক

**প্রশ্ন ৬: ডিফল্ট ভ্যালু সহ prop-এ `undefined`, `null`, বা `false` পাঠালে কী হবে?**  
উত্তর: শুধুমাত্র `undefined` বা prop না পাঠালে ডিফল্ট ভ্যালু ব্যবহৃত হবে। `null` বা `false` পাঠালে সেই মানই ব্যবহৃত হবে, ডিফল্ট নয়।

**প্রশ্ন ৭: কিভাবে এক কম্পোনেন্ট থেকে আরেক কম্পোনেন্টে সব props ফরওয়ার্ড করা যায়?**  
উত্তর: Spread syntax ব্যবহার করে: `<Avatar {...props} />`

**প্রশ্ন ৮: ফাংশনকে prop হিসেবে পাঠানো যায় কি? উদাহরণ দিন।**  
উত্তর: হ্যাঁ। উদাহরণ: `<Button onClick={handleClick} />` যেখানে `handleClick` একটি ফাংশন।

**প্রশ্ন ৯: props ফরওয়ার্ড করার জন্য spread syntax ব্যবহারে সতর্ক থাকতে কেন বলা হয়?**  
উত্তর: কারণ এতে কোন কোন prop পাঠানো হচ্ছে তা স্পষ্ট থাকে না এবং কোড বোঝা কঠিন হতে পারে।
