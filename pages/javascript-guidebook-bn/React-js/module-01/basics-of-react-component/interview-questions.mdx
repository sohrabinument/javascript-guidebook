# রিয়্যাক্ট কম্পোনেন্ট – ইন্টারভিউ প্রশ্ন ও উত্তর

এই প্রশ্নোত্তরগুলো রিয়্যাক্ট কম্পোনেন্ট সম্পর্কে ইন্টারভিউ প্রস্তুতি বা নিজের ধারণা যাচাইয়ের জন্য ব্যবহার করুন।

---

**প্রশ্ন ১: রিয়্যাক্ট কম্পোনেন্ট কী?**  
উত্তর: রিয়্যাক্টের একটি পুনঃব্যবহারযোগ্য, স্বয়ংসম্পূর্ণ UI অংশ, যা ফাংশন বা ক্লাস হিসেবে সংজ্ঞায়িত হয়।

**প্রশ্ন ২: কম্পোনেন্টের নাম বড় হাতের অক্ষরে শুরু করা কেন জরুরি?**  
উত্তর: রিয়্যাক্ট ছোট হাতের ট্যাগকে HTML এলিমেন্ট এবং বড় হাতের ট্যাগকে কাস্টম কম্পোনেন্ট হিসেবে ধরে।

**প্রশ্ন ৩: রিয়্যাক্টে ফাংশন কম্পোনেন্ট কীভাবে তৈরি করবেন?**  
উত্তর: বড় হাতের অক্ষরে শুরু হওয়া একটি জাভাস্ক্রিপ্ট ফাংশন লিখে, যা JSX রিটার্ন করে।

**প্রশ্ন ৪: এক কম্পোনেন্টকে আরেকটিতে কীভাবে ব্যবহার করবেন?**  
উত্তর: ইম্পোর্ট করে কাস্টম ট্যাগ হিসেবে ব্যবহার করুন, যেমন `<Profile />`।

**প্রশ্ন ৫: রিয়্যাক্টে কম্পোনেন্ট ব্যবহারের সুবিধা কী?**  
উত্তর: পুনঃব্যবহারযোগ্যতা, দায়িত্ব বিভাজন, সহজ রক্ষণাবেক্ষণ ও ভালো সংগঠন।

**প্রশ্ন ৬: কম্পোনেন্টের ভিতরে ফাংশন ডিফাইন করা যাবে কি? কেন নয়?**  
উত্তর: না, কম্পোনেন্টের ভিতরে ফাংশন ডিফাইন করা উচিত নয়। হেল্পার ফাংশন বাইরে বা আলাদা কম্পোনেন্ট হিসেবে লিখুন—এতে কোড পরিষ্কার ও রক্ষণাবেক্ষণযোগ্য থাকে।

**প্রশ্ন ৭: পরিস্থিতি: একটি বাটন বিভিন্ন জায়গায় বিভিন্ন লেবেল নিয়ে ব্যবহার করতে হবে। কীভাবে পুনঃব্যবহারযোগ্য কম্পোনেন্ট বানাবেন?**  
উত্তর: একটি Button কম্পোনেন্ট বানিয়ে label প্রপস নিন এবং যেখানে দরকার সেখানে ভিন্ন ভিন্ন label পাঠান।

**প্রশ্ন ৮: কম্পোনেন্টের নাম বড় হাতের অক্ষরে না লিখলে কী হয়?**  
উত্তর: রিয়্যাক্ট এটিকে সাধারণ HTML ট্যাগ হিসেবে ধরে, ফলে আপনার কম্পোনেন্ট প্রত্যাশিতভাবে রেন্ডার হবে না।

**প্রশ্ন ৯: কখন একটি কম্পোনেন্ট আলাদা ফাইলে রাখা উচিত?**  
উত্তর: যখন কম্পোনেন্ট বড়, জটিল বা একাধিক জায়গায় ব্যবহৃত হয়।

---

**টিপ:** এই প্রশ্ন-উত্তরগুলো পড়ুন এবং প্রতিটি ধারণা নিজের ভাষায় ব্যাখ্যা করার চেষ্টা করুন—এটাই সেরা ইন্টারভিউ প্রস্তুতি!
