## রিয়্যাক্ট কম্পোনেন্ট কী?

একটি **কম্পোনেন্ট** হলো রিয়্যাক্টের একটি পুনঃব্যবহারযোগ্য, স্বয়ংসম্পূর্ণ UI অংশ। কম্পোনেন্ট হচ্ছে রিয়্যাক্ট অ্যাপ্লিকেশনের বিল্ডিং ব্লক—আপনার UI-এর প্রতিটি অংশ, একটি বাটন থেকে শুরু করে পুরো পেজ পর্যন্ত, কম্পোনেন্ট হতে পারে। কম্পোনেন্ট ব্যবহার করে জটিল ইন্টারফেসকে ছোট, সহজে ব্যবস্থাপনাযোগ্য ও পুনঃব্যবহারযোগ্য অংশে ভাগ করা যায়।

### কেন কম্পোনেন্ট ব্যবহার করবেন?

- **পুনঃব্যবহারযোগ্যতা:** একবার লিখে অ্যাপের যেকোনো জায়গায় ব্যবহার করুন।
- **দায়িত্ব বিভাজন:** প্রতিটি কম্পোনেন্ট নিজের লজিক ও UI পরিচালনা করে।
- **রক্ষণাবেক্ষণ সহজ:** আপডেট, ডিবাগ ও টেস্ট করা সহজ।

### কম্পোনেন্টের ধরন

- **ফাংশন কম্পোনেন্ট:** সবচেয়ে প্রচলিত ও আধুনিক উপায়। এগুলো জাভাস্ক্রিপ্ট ফাংশন যা JSX রিটার্ন করে।
- **ক্লাস কম্পোনেন্ট:** পুরনো পদ্ধতি, এখন কম ব্যবহৃত, ES6 ক্লাস ব্যবহার করে।

## ফাংশন কম্পোনেন্ট তৈরি করা

```jsx
export default function Profile() {
  return <img src="https://i.imgur.com/MK3eW3Am.jpg" alt="Katherine Johnson" />;
}
```

**মূল বিষয়:**

- ফাংশনের নাম অবশ্যই বড় হাতের অক্ষরে শুরু করতে হবে।
- ফাংশনটি JSX (মার্কআপ) রিটার্ন করে।
- অন্যত্র ব্যবহার করতে চাইলে `export default` ব্যবহার করুন।

## কম্পোনেন্ট ব্যবহার

একটি কম্পোনেন্ট অন্য কম্পোনেন্টের ভিতরে কাস্টম HTML ট্যাগের মতো ব্যবহার করা যায়:

```jsx
import Profile from "./Profile";

export default function Gallery() {
  return (
    <section>
      <h1>অসাধারণ বিজ্ঞানীরা</h1>
      <Profile />
      <Profile />
      <Profile />
    </section>
  );
}
```

## সেরা চর্চা ও নিয়ম

- **কম্পোনেন্টের নাম বড় হাতের অক্ষরে শুরু করুন।**
- **কম্পোনেন্ট ফাংশন থেকে JSX রিটার্ন করুন।**
- **কম্পোনেন্টের ভিতরে আরেকটি ফাংশন ডিফাইন করবেন না (হেল্পার ফাংশন বাইরে বা আলাদা কম্পোনেন্ট হিসেবে লিখুন)।**
- **ছোট ও পুনঃব্যবহারযোগ্য নয় এমন কম্পোনেন্ট একই ফাইলে রাখা যায়। বড় বা পুনঃব্যবহারযোগ্য হলে আলাদা ফাইলে রাখুন।**
