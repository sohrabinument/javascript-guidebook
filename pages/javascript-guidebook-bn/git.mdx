# গিট এবং গিটহাব রিফ্রেশার

## গিট কি?

গিট হলো একটি ভার্সন কন্ট্রোল সিস্টেম যা আপনার প্রজেক্টের কোড ম্যানেজ করতে সাহায্য করে। এর মাধ্যমে আপনি:

- কোডের পরিবর্তন ট্র্যাক করতে পারবেন
- টিমের সাথে কাজ করতে পারবেন
- কোডের পূর্ববর্তী ভার্সনে ফিরে যেতে পারবেন
- বিভিন্ন ফিচার নিয়ে পৃথক পৃথক কাজ করতে পারবেন

## গিটহাব কি?

গিটহাব হলো একটি ওয়েব-ভিত্তিক প্ল্যাটফর্ম যেখানে আপনি আপনার গিট রিপোজিটরি হোস্ট করতে পারেন। এটি আপনাকে:

- কোড শেয়ার করতে
- অন্যদের সাথে কোলাবোরেট করতে
- প্রজেক্ট ম্যানেজ করতে
- ওপেন সোর্স প্রজেক্টে কন্ট্রিবিউট করতে সাহায্য করে

## মৌলিক গিট কমান্ডস

```bash
git init            # নতুন রিপোজিটরি শুরু করা
git add .           # সব ফাইল স্টেজিং এ যোগ করা
git commit -m "মেসেজ"  # পরিবর্তন সেভ করা
git push            # রিমোট রিপোজিটরিতে পাঠানো
git pull            # রিমোট থেকে আপডেট নেওয়া
git branch          # ব্রাঞ্চ দেখা
git checkout        # ব্রাঞ্চ পরিবর্তন করা
```

## গিট এবং গিটহাব

## গিট কি ?

গিট হলো একটি `ভার্সন কনট্রোল` টুল।

ভার্সন কনট্রোল কি ?

ধরুন আপনি আপনার কম্পিউটার এ কোন ডকুমেন্ট লিখছেন,সেখানে হয়তো আজ কিছু লিখছেন আবার হয়তো অন্যদিন কিছু লিখছেন। এভাবে হয়তো আপনার কখনও মনে হলো যে আপনি গত দুইদিন যা লিখেছেন তা হয়তো সঠিক নয়, আপনি তা ডিলিট করে আবার দুই দিন আগে যা লিখেছেন সেই অবস্থায় ফিরত যেতে চান। এক্ষেত্র আপনাকে যেটা করা লাগবে তা হলো এই দুইদিনের সমস্ত লিখা মুছে ফেলতে হবে। এখন এই অবস্থায় আপনার যদি মনে না থাকে যে আপানর ঠিক কোন পর্যন্ত মুছা উচিত বা দুইদিন আগে ঠিক কতটুকু লেখা হয়েছিলো,তখন আপনি কি করবেন? ঠিক এই কাজটায় গিট হেল্প করে থাকে। গিট কোন ফাইলের প্রতিটা চেঞ্জেস গুলোকে মনিটরিং করে বা ট্র্যাক রাখে এবং প্রতিটা চেঞ্জেস গুলোকে ভার্সনাইজ করে। যাতে আপনি যেকোন সময় যেকোনো ভার্সনে সুইচ করতে পারেন ।

## গিটের কিছু প্রয়োজনীয় কমান্ড যা আমাদের গিটের লোকাল রিপজিটরিতে ফাইল ম্যানেজ করতে প্রয়োজন হবে।

1. **গিট ইনিশিয়ালাইজ করা**

```bash
 git init
```

2. **গিট এর স্ট্যাটাস চেক করা**

```bash
git status
```

3. **ফাইলকে গিটের ট্রাক করার জন্য স্টেজ এ উঠানো**

```bash
git add <filename> //For stage any specific file
or
git add --all // if you want to add all files
or
git add .  // if you want to add all files in the current working directory
```

4. **কমিট করা**

```bash
git commit -m "<commit massage>"
```

5. **গিট লগ চেক করা**

```bash
git log
or
git log --oneline
```

6. **একটা কমিট থেকে অন্য কমিটের অবস্থায় ফিরে যাওয়া**

```bash
git reset --hard <commit id>
```

7. **গিটের প্রতিটা চেঞ্জের রেফারেন্স সহ লগ দেখার জন্য**

```bash
git reflog
```

8. **গিটের ট্র্যাক থেকে কোন ফাইল রিমূভ করা**

```bash
git rm <filename>
```

9. **ব্রাঞ্চ তৈরী করা**

```bash
git branch <branch name eg: bugfix/fixname >
```

10. **ব্রাঞ্চ লিস্ট দেখা**

```bash
git branch --list
```

11. **এক ব্রাঞ্চ থেকে অন্য ব্রাঞ্চ এ সুইচ করা**

```bash
git switch <branch name >
```

12. **এক ব্রাঞ্চের কোড অন্য ব্রাঞ্চে মার্জ করা**

  <Callout type='warning' emoji="💡">
    এক্ষেত্রে যেই ব্রাঞ্চে অন্য ব্রাঞ্চের কোড আনতে হবে,প্রথমে `git switch <branchname>` করে প্রথমে সেই ব্রাঞ্চে যেতে হবে। তারপর কমান্ড দিতে হবে।
</Callout>

```bash
git merge <branch name >
```

13. **গিট থেকে ব্রাঞ্চ ডিলিট করা**

```bash
git branch -d <branch name >
```

14. **ব্রাঞ্চ নেম পরিবর্তন করতে চাইলে**

 <Callout type='warning' emoji="💡"> এক্ষেত্রে যেই ব্রাঞ্চের নাম পরিবর্তন করতে হবে আমাদেরকে প্রথমে `git switch <branchname>` করে প্রথমে সেই ব্রাঞ্চে যেতে হবে। তারপর কমান্ড দিতে হবে। </Callout>

```bash
git branch -m <updated branch name >
```

15. **গিট স্ট্যাস করা (আনকমিটেড কাজ গিটের ড্রাফট এ সেভ রাখা)**

```bash
git stash
```

16. **গিট স্ট্যাস লিস্ট চেক করা**

```bash
git stash list
```

17. **গিটের স্ট্যাস এ কি কি সেভ রাখা হয়েছিল তা দেখতে**

```bash
git stash show -p
```

18. **গিটের স্ট্যাস এপ্লাই করার জন্য**

```bash
git stash apply <statsh id>
or
git stash pop
```

19. কোন ফাইল ভুলবশত গিটে এড করে ফেললে বা কমিট করার পর যদি মনে হয় ফাইলটা গিট থেকে ইগনোর করতে হবে তাহলে:

<Steps>

### ১ম স্টেপঃ

একটা .gitignore ফাইল বানাতে হবে এবং তাতে উক্ত ফাইলের নাম এড করা লাগবে ।

### ২য় স্টেপঃ

```bash
git rm --cached <filename>
```

</Steps>

20. গিটহাব বা রিমোট রিপোজিটরিতে ফাইল এড করা বা পুশ করার জন্যঃ

<Steps>
### ১ম স্টেপঃ

```bash
git remote add <origin name eg: orgin or something > <github repository link>
```

example:

`git remote add origin git@github.com:Deveripon/reactive-accelarator.git`

### ২য় স্টেপঃ

```bash
git push -u <origin name> <branch name>
```

example:

`git push -u origin main`

</Steps>

21. রিমোট রিপোজিটরি থেকে পুল করা

```bash
git pull
```
